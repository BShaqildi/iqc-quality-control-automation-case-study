# User Stories

## Story 1 – Auto-Populated Loan Data
As a QC analyst, I want loan data to automatically populate so that I do not need to manually enter information.

### Acceptance Criteria
- Loan data is pulled via API
- No manual entry required
- Data reflects latest loan status
- QC review can begin immediately

---

## Story 2 – Error Citation Workflow
As a QC analyst, I want to cite errors against loan stakeholders so that issues can be tracked and resolved.

### Acceptance Criteria
- User can assign errors to processors, underwriters, or closers
- Errors include category and severity
- Errors trigger notifications
- Errors remain tracked until resolved

---

## Story 3 – Automated QC Checks
As a QC analyst, I want automated validations to run so that I can focus on higher-value review tasks.

### Acceptance Criteria
- System runs predefined QC checks
- Flags potential issues automatically
- Reduces need for manual validation
